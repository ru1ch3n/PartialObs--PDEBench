How this site is built

All paper metadata and summaries live as YAML under data/papers/. The Python generator reads YAML and writes the static website into docs/ (GitHub Pages).

Current coverage: 46 curated pages out of 300 indexed papers.

Add a new paper (or curate an existing one)

  1. Create / edit a YAML file in data/papers/. One paper = one YAML file. Use a short, URL-friendly filename (slug), e.g. fno.yaml or diffusionpde.yaml.
  2. Fill the key fields: title, year, method class, links, and (ideally) PDEs/tasks/experiments/baselines.
  3. Regenerate the site: run python scripts/generate_research_site.py from the repo root.
  4. Preview locally (optional): python -m http.server -d docs 8000 and open the shown URL.
  5. Commit + push your changes (data/papers/*.yaml + regenerated docs/) and open a PR.

YAML schema (human-friendly)

Write plain text; the site will format it. You can include LaTeX in core_math and theory items.

# Paper summary template (YAML)
#
# Copy this file to: data/papers/<slug>.yaml
# Then fill in fields. The website generator reads *all* YAML files in
# data/papers/ and builds docs/.
#
# Tips
# - Keep lists short and specific.
# - Put the most important equation(s) in core_math (LaTeX, no $...$ needed).
# - If you don't know a field yet, leave it empty.

slug: example-paper-slug  # file name without .yaml is used if omitted

# Bibliography
short_title: ExampleMethod
full_title: Full paper title goes here
authors: First Author, Second Author, …
year: 2025
venue: arXiv / NeurIPS / ICLR / JCP / …

# Optional: lightweight classification used in filters/tables
category: Operator learning / PINN / Diffusion / Theory / Survey / …
method_class: Operator learning  # Operator learning | Diffusion | PINN / physics-constrained | Graph / mesh | Transformers | Benchmark | SciML
status: index  # index (placeholder) | curated (has full summary)

links:
  paper: https://arxiv.org/abs/XXXX.XXXXX
  paper_label: arXiv:XXXX.XXXXX
  code: https://github.com/...
  project: https://...

# Short site-level summary
tldr: >-
  1–3 sentences. What problem is solved, what method is introduced, and what is the headline result?

# “What is new?” bullets
contrib:
  - Main contribution #1
  - Main contribution #2

# Optional: theoretical contributions (with assumptions/conditions)
theory:
  - Main theorem / guarantee / analysis result #1
  - Main theorem / guarantee / analysis result #2

# Core method in math (LaTeX). Keep it minimal and correct.
core_math:
  - "\\min_\\theta \\; \\mathbb{E}_{(x,y)}\\, \\|f_\\theta(x) - y\\|^2"
  - "\\text{s.t. }\\; \\mathcal{N}(f_\\theta) = 0"

# Experiments (fill what the paper *actually* runs)
pdes:
  - Burgers
  - Navier–Stokes
tasks:
  - Forward surrogate / operator learning
  - Inverse problem / partial observation reconstruction

setting:
  - Data: (dataset name) / simulation / PDEBench / custom
  - Observation: sparse sensors / random mask / block mask / noisy
  - Metrics: relative L2 / RMSE / NLL / CRPS / …
  - Train/test: (split description)

baselines:
  - FNO
  - PINO
  - DiffusionPDE

# Optional: results tables (use for one or two key tables only)
results_tables:
  - title: Main table title
    note: Optional short note (e.g., “lower is better”, “copied from Table 2”).
    header: [Method, Metric1, Metric2]
    rows:
      - [Ours, "0.12", "0.08"]
      - [Baseline, "0.20", "0.15"]

# Optional: any replication warning
benchmark_note: ""

Results tables

To include a paper's main quantitative results, add results_tables entries in YAML:

results_tables:
  - title: Main table (as reported)
    note: Optional short note / dataset / metric.
    header: [Model, Error \u2193, Speed]  # columns
    rows:
      - [FNO, 0.012, 1\u00d7]
      - [Your method, 0.008, 0.9\u00d7]
Please copy numbers exactly from the paper (and mention the setting in note).

Good curation checklist

  • Core method: add 1–3 key equations in core_math.
  • Theory: list theorems/guarantees/assumptions in theory.
  • PDEs & tasks: name the PDEs used (Navier–Stokes, Burgers, Darcy, etc.) and the task type (forecasting, inverse, UQ...).
  • Partial observation: describe the mask (sensors, missing pixels, sparse trajectories, etc.).
  • Baselines: list compared methods and training setup differences.
